# Claude Code 메모리 시스템

## 개요
Claude Code는 대화가 끝나면 컨텍스트가 초기화된다.
하지만 **메모리 시스템**을 사용하면 reload/새 대화 후에도 핵심 정보를 유지할 수 있다.

## 저장 위치
```
~/.claude/projects/<project-path>/memory/
├── MEMORY.md          # 인덱스 파일 (메모리 목록)
├── user_role.md       # 개별 메모리 파일 예시
├── feedback_testing.md
└── ...
```

## 메모리 타입

| 타입 | 용도 | 예시 |
|------|------|------|
| **user** | 사용자 역할, 선호도, 지식 수준 | "백엔드 개발자, Python 숙련" |
| **feedback** | 작업 방식 피드백 (DO/DON'T) | "테스트에서 DB mock 쓰지 마라" |
| **project** | 진행 중 작업, 목표, 마감일 | "3/28까지 인증 모듈 리팩토링" |
| **reference** | 외부 리소스 위치 | "버그 트래킹은 Linear INGEST 프로젝트" |

## 메모리 파일 형식
각 메모리는 개별 `.md` 파일로 저장되며 frontmatter를 포함한다.

```markdown
---
name: 사용자 역할
description: 사용자의 역할과 기술 스택 정보
type: user
---

백엔드 개발자. Python, FastAPI 사용. 산업 데이터 분석 도메인.
```

## MEMORY.md (인덱스)
`MEMORY.md`는 메모리 파일들의 목록만 담는 인덱스 파일이다.
메모리 내용 자체를 여기에 쓰지 않는다.

```markdown
- [user_role.md](user_role.md) — 사용자 역할과 기술 스택
- [feedback_testing.md](feedback_testing.md) — 테스트 작성 시 주의사항
```

## 사용법

### 저장 요청
대화 중 아래처럼 요청하면 된다:
- "이거 기억해둬" → 적절한 타입으로 메모리 저장
- "내가 백엔드 개발자라는 거 기억해" → user 타입으로 저장
- "테스트에서 mock 쓰지 말라고 했잖아, 기억해둬" → feedback 타입으로 저장

### 삭제 요청
- "그 메모리 지워줘" → 해당 메모리 파일 삭제 + 인덱스 업데이트

### 자동 동작
- 새 대화 시작 시 `MEMORY.md` 인덱스가 자동으로 로드됨
- 관련 메모리가 있으면 Claude가 참고하여 응답

## 저장하지 않는 것
- 코드 구조, 파일 경로 → 직접 탐색 가능
- git 히스토리 → `git log`로 확인 가능
- 디버깅 솔루션 → 코드와 커밋 메시지에 이미 존재
- CLAUDE.md에 이미 문서화된 내용
- 현재 대화에서만 쓰이는 임시 정보

## 주의사항
- `MEMORY.md`는 200줄 이후 잘리므로 인덱스는 간결하게 유지
- 메모리는 시간이 지나면 stale해질 수 있음 → 실제 코드/리소스와 교차 검증 필요
- 중복 메모리 금지 → 기존 메모리 업데이트 우선
