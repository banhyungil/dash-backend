# Excel 데이터 화면 표시 방안

> 작성일: 2026-03-24
> 상태: 검토 대기

---

## 1. 현황 분석

### 현재 화면에 표시되는 정보
| 구분 | 내용 |
|------|------|
| KPI 카드 | 총 사이클, 가동시간, 활성 롤러, 평균 RPM, 평균/최대 MPM, Burst 수, Peak Impact 수 |
| RPM 차트 | 시간축 MPM 시각화 (6:00~20:00), 장치별 색상 구분 |
| 진동 차트 | Pulse 가속도(X,Y,Z) + VIB 가속도(X,Z), 고진동 이벤트 마커 |
| 사이클 상세 모달 | 개별 사이클 클릭 시 파형 표시 |

### Excel에만 존재하는 정보 (화면 미표시)

#### Sheet 1: Timeline (분 단위 RPM 테이블)
- 시간(HH:MM) 기준 R1~R4 노드별 평균 RPM
- 데이터 갭(공회전 구간) 표시
- 시간 점프 감지

#### Sheet 2~5: Vib_R1 ~ Vib_R4 (노드별 진동 상세 테이블)
- 사이클별 행: Date, Time, Cycle, RPM, PX_RMS, PY_RMS, PZ_RMS, VX_RMS, VZ_RMS
- RPM 구간별 통계: RPM Range(10단위), Count, Min, Max, Average

## 2. 방안

### 방안 A: 탭 추가 방식 (기존 차트 탭 옆에 테이블 탭 추가)

현재 ChartsPage에 `RPM | RPM 3Panel | Vibration | Vib 3Panel` 탭이 있으므로, 여기에 테이블 탭을 추가한다.

```
[RPM] [RPM 3Panel] [Vibration] [Vib 3Panel] [📋 Timeline] [📋 진동 상세]
```

**Timeline 탭**
- 분 단위 테이블: 시간 | R1 | R2 | R3 | R4
- 갭 구간은 배경색으로 강조
- 스크롤 가능한 테이블, 헤더 고정

**진동 상세 탭**
- 노드 선택 서브탭: R1 | R2 | R3 | R4
- 사이클별 테이블: Date | Time | Cycle | RPM | PX_RMS | PY_RMS | PZ_RMS | VX_RMS | VZ_RMS
- 하단에 RPM 구간별 통계 요약 테이블
- 고진동 이벤트 행은 배경색 강조

**장점:** 기존 UI 패턴과 일관성 유지, 구현 간단
**단점:** 탭이 많아져 복잡해질 수 있음

---

### 방안 B: 별도 페이지 방식

사이드바에 새 메뉴를 추가한다.

```
📊 일일 차트    → /
📋 상세 데이터  → /detail
📁 데이터 관리  → /manager
```

`/detail` 페이지에 Timeline 테이블과 진동 상세 테이블을 배치한다.

**장점:** 차트 페이지가 깔끔하게 유지됨
**단점:** 날짜 선택 상태 공유 필요 (이미 Zustand 사용 중이라 문제없음), 페이지 이동 필요

---

### 방안 C: 차트 아래 접이식 패널 (Collapsible)

차트 영역 아래에 접이식 테이블 섹션을 배치한다.

```
┌─────────────────────────┐
│  KPI 카드                │
├─────────────────────────┤
│  [탭] 차트 영역          │
├─────────────────────────┤
│  ▶ Timeline 테이블       │  ← 클릭하면 펼침
├─────────────────────────┤
│  ▶ 진동 상세 테이블      │  ← 클릭하면 펼침
└─────────────────────────┘
```

**장점:** 한 페이지에서 차트와 테이블 모두 확인 가능
**단점:** 스크롤이 길어질 수 있음

---

## 3. 추천 방안

**방안 A (탭 추가)** 를 추천한다.

이유:
1. 기존 탭 구조와 자연스럽게 통합
2. 라우팅 변경 불필요
3. 사용자가 차트↔테이블 간 빠르게 전환 가능
4. 구현 범위가 가장 작음

## 4. 구현 계획 (방안 A 기준)

### 4.1 백엔드 변경

현재 `GET /cycles/daily` 응답에 이미 cycles 배열이 포함되어 있으므로, 프론트에서 가공하여 표시할 수 있다. **백엔드 API 추가 불필요.**

다만, Timeline 분 단위 집계 로직이 `excel_export.py`의 `_build_timeline_sheet()`에 있으므로:

- **선택지 1)** 프론트엔드에서 cycles 데이터로 직접 분 단위 집계 (추천)
  - cycles 데이터에 timestamp, session, rpm_mean이 이미 있으므로 가능
  - Excel과 동일한 로직을 JS로 구현
- **선택지 2)** 백엔드에 Timeline JSON API 추가
  - 데이터 양이 많아 성능 이슈가 우려될 경우

### 4.2 프론트엔드 변경

| 작업 | 파일 | 설명 |
|------|------|------|
| Timeline 테이블 컴포넌트 | `components/TimelineTable.tsx` (신규) | 분 단위 RPM 테이블, 갭 강조 |
| 진동 상세 테이블 컴포넌트 | `components/VibDetailTable.tsx` (신규) | 노드별 사이클 RMS 테이블 + RPM 구간 통계 |
| ChartsPage 탭 추가 | `pages/ChartsPage.tsx` (수정) | 탭 목록에 Timeline, 진동 상세 추가 |
| 데이터 가공 유틸 | `utils/dataTransform.ts` (신규) | 분 단위 집계, RPM 구간 통계 계산 |

## 5. 고려사항

- **성능**: 사이클 수가 많을 경우(수천 건) 테이블 렌더링 성능 → 가상 스크롤(react-virtual) 도입 검토
- **정렬/필터**: 테이블 컬럼 클릭 시 정렬 기능 추가 고려
- **Excel과 일관성**: 화면 테이블과 Excel 출력 데이터가 동일하도록 검증 필요
- **반응형**: 컬럼이 많으므로 가로 스크롤 또는 컬럼 토글 기능 고려
